# セットアップガイド

このガイドでは、各スクリプトの初期設定手順を説明します。
スクリプトエディタの基本操作も含めているため、GAS初心者の方でも設定できます。

---

## 共通手順：スクリプトエディタを開く

1. Google スプレッドシートを開く
2. メニューから **「拡張機能」→「Apps Script」** をクリック
3. スクリプトエディタが別タブで開きます

---

## ① 期限管理リマインダー（deadline-reminder.gs）

### 必要なもの
- Slack Incoming Webhook URL（Slack の管理者に発行してもらうか、自分で作成）

### 設定手順

**Step 1: スクリプトをコピー**
1. スクリプトエディタに `deadline-reminder.gs` の内容を貼り付ける
2. 保存（Ctrl+S）

**Step 2: 初期設定を実行**
1. 関数の選択プルダウンから `initializeSettings` を選択
2. 「実行」ボタンをクリック
3. 初回は権限の承認を求められるので「許可」をクリック

**Step 3: スクリプトプロパティを更新**
1. メニュー「プロジェクトの設定」→「スクリプトプロパティ」を開く
2. `DEADLINE_SLACK_WEBHOOK_URL` の値を実際の Webhook URL に変更
3. `DEADLINE_NOTIFY_DAYS` は通知したい日数をカンマ区切りで設定（例: `10,3,0`）

**Step 4: トリガーを設定**
1. 左サイドバーの「トリガー」（時計アイコン）をクリック
2. 「トリガーを追加」をクリック
3. 以下のように設定する：
   - 実行する関数: `checkDeadlines`
   - イベントのソース: 時間主導型
   - 時間ベースのトリガーのタイプ: 日付ベースのタイマー
   - 時刻: 午前9時〜10時

**Step 5: スプレッドシートの準備**
- A列: 会社名
- B列: 期限日（`2026/04/30` などの日付形式）

## ② フォーム自動振り分け（form-router.gs）

### 必要なもの
- Google フォームと連携済みのスプレッドシート

### 設定手順

**Step 1: スクリプトをコピー＆初期設定**
1. フォームの回答が記録されるスプレッドシートでスクリプトエディタを開く
2. `form-router.gs` の内容を貼り付けて保存
3. `initializeSettings` を実行（「設定」シートが自動作成される）

**Step 2: 部署の追加**
- 「設定」シートのA列2行目以降に部署名を入力する
- 例: `営業部`、`総務部`、`人事部`
- 追加した部署名と同名のシートが自動作成されます

**Step 3: トリガーを設定**
1. 「トリガーを追加」をクリック
2. 以下のように設定する：
   - 実行する関数: `onFormSubmit`
   - イベントのソース: スプレッドシートから
   - イベントの種類: フォーム送信時

**Step 4: 動作確認**
- テスト用のフォーム回答を送信して、対応する部署シートに転記されることを確認

## ③ AI顧客レビュー自動分析（ai-review-analyzer.gs）

### 必要なもの
- Gemini API キー（無料で取得可能）

### API キーの取得方法
1. [Google AI Studio](https://aistudio.google.com) にアクセス
2. Googleアカウントでサインイン
3. 「Get API key」→「APIキーを作成」をクリック
4. 生成されたAPIキーをコピーしておく

### 設定手順

**Step 1: スクリプトをコピー＆初期設定**
1. スクリプトエディタに `ai-review-analyzer.gs` の内容を貼り付けて保存
2. `initializeSettings` を実行（ヘッダー行が自動設定される）

**Step 2: APIキーを設定**
1. メニュー「プロジェクトの設定」→「スクリプトプロパティ」を開く
2. `GEMINI_API_KEY` の値を取得したAPIキーに変更する
   > ⚠️ APIキーはスクリプトプロパティで管理し、コードに直接記載しないこと

**Step 3: レビューを入力して分析実行**
1. スプレッドシートのA列（2行目以降）に顧客レビューを入力
2. スクリプトエディタで `analyzeAllReviews` を実行
3. B列に感情分類（ポジティブ／ネガティブ／中立）、C列に要約が自動入力される

## よくある質問

**Q: 「このアプリは確認されていません」と表示される**
A: 自分で作成したスクリプトのため表示されます。「詳細」→「安全ではないページに移動」をクリックして進んでください。

**Q: スクリプトプロパティはどこで確認できますか？**
A: スクリプトエディタの左サイドバー「プロジェクトの設定（歯車アイコン）」→「スクリプトプロパティ」から確認・編集できます。

**Q: API キーが漏れないか心配です**
A: スクリプトプロパティはスプレッドシートを共有しても公開されません。コードにAPIキーを直接書くことだけ避けてください。
